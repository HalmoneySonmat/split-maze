초록 (Abstract)

이질적인 두 네트워크 — procgen 미로를 푸는 IMPALA-CNN RL 에이전트(결정자)와 합성 미로언어를 말하는 소형 트랜스포머 LM(해석자) — 를 모두 from scratch로 공동 학습하고, 그 사이에 분리된 재구성 loss로 학습되는 위치 무관 coactivation 매핑(ACC, "인공 뇌량")을 두었다. 핵심 가설: 분리-재구성 ACC(V2)가 next-token 어댑터(B4, = SPLIT-9 패턴)보다 에이전트의 실제 내부 목표를 더 충실히 읽는다.

목표 오일반화 미로에서의 결정적 테스트에서 가설은 기각됐다(Scenario C): V2의 충실도(per-slot 평균 0.375, swap-following 0.42)가 B4(0.66, 0.83)보다 낮다. 그러나 세 가지 풍부한 negative result를 얻었다 — ① 목표 오일반화가 표상 수준에서 직접 읽힌다(모든 빌드 cheese_dir 충실도 in-dist 0.85 → OOD 0.07 붕괴), ② "충실 ≠ 합리화" 재프레임: B4의 OOD "합리화"(50%)는 둘러대기가 아니라 목표 오일반화한 에이전트를 충실히 읽은 결과임을 활성 스왑이 인과적으로 증명, ③ ACC의 단일 요약-벡터 병목이 어댑터의 분산 cross-attention 주입보다 충실도가 낮다는 아키텍처 통찰. 후속 통제 2×2 실험(인터페이스 × loss)이 이를 확증한다 — 인터페이스를 동일하게 맞춰도 재구성이 next-token보다 약하므로, V2의 열세는 인터페이스 아티팩트가 아니라 학습 신호 자체의 성질이다(§3.4).

후속 (Phase 5 — CCM). V2 기각 이후, 다리를 학습하지 않고 두 망이 같은 장면을 볼 때 켜진 노드의 대응을 기록(기억)만 해 에이전트→LM을 구동하는 공동활성 기억 뇌량(CCM)을 탐구했다 (backprop 없음; W는 닫힌형 통계 버퍼). 세 결과 — ① 기록만으로 에이전트 표상이 인과적으로 전이된다(swap-following 0.445 = 학습 어댑터 B4의 52%); ② 결정 성분은 "공통모드(평균) 제거"다 — 날것 공동활성(Hebbian 외적)은 붕괴하나 중심화 또는 화이트닝 하나만으로 거의 완전 복원(중복 경로)되어 "다리는 무엇이 같이 켜졌나가 아니라 무엇이 다른가를 기록해야 함"을 보인다; ③ 두 뇌가 적응하며 다리가 자라는 닫힌고리는 *기록 W · 에이전트만* 으로는 미지지(loss↓≠성공)이나, 다리를 가소성으로 풀고(기억→씨앗) 두 뇌가 마중하면(step3: agent gentle + LM blocks[0] + W) 얇은 다리가 수렴 번역기 천장(swap 0.78)까지 자란다 — task · 언어 보존, 공동적응 기여는 modest(~+0.06)이나 W-무관(기록 ridge 0.445→0.510)하여 실재. 사용자 원래 비전("두 뇌가 동시에 적응하며 다리가 자란다")의 지지 증거(§3.5; 5-seed 절차 확정 — 효과 +0.064±0.020, 5/5 양성). 나아가 다른 결정자 뇌 3개로의 일반화도 확인(GENERALIZES — 평균 +0.081, 3/3 양성; 해석자 LM 공유라 결정자 뇌 한정, 효과는 뇌-의존 — 과제가 붕괴한 1개 뇌에선 사라짐).

1. 서론 (Introduction)

SPLIT-9 (선행 negative result). 이미 학습된 Go-Net과 동결 사전학습 LLM을 사후 어댑터로 묶고 next-token loss 하나로 학습했더니, loss 개선의 ~95%가 LLM의 도메인 prior, 보드별 신호의 인과 기여는 ~5%, 정성 평가 10/10 오답이었다. 진짜 원인은 어댑터가 아니라 동결 LLM의 prior 자체 — 분리뇌 환자의 좌반구가 만드는 그럴듯한 합리화 함정이다.

SPLIT-MNIST (선행 검증). 가장 작은 toy(좌/우로 가른 MNIST + 두 CNN + ACC, 전부 from scratch)에서 분리된 재구성 loss(V2/γ)는 한쪽 자극만으로 반대쪽 표현을 충실히 복원했고(cosine 0.81), next-token only(B4 = SPLIT-9 패턴)는 유의하게 약했다(Δ+0.16, p<0.0001). 교훈: 빠진 조각은 공동 학습 자체가 아니라 학습 신호의 분리다.

SPLIT-MAZE (본 연구). 검증된 V2 패턴을 이질적인 진짜 환경에서, 양쪽 다 from scratch로 재검증한다. 목표 오일반화(Langosco et al., 2022 — 학습 중 치즈가 항상 우상단에 있던 미로 에이전트가, 테스트에서 치즈가 다른 곳이어도 우상단으로 향함)는 충실한 해석 vs 합리화를 가르는 완벽한 판별기다. 합리화하는 해석자는 "치즈 찾는 중"(학습 prior)이라 말하고, 충실한 해석자는 "우상단으로 향함"(실제 내부 목표)이라 말해야 한다. 정답을 아는 환경에서 SPLIT-9의 실패 양식을 측정한다.

2. 설정 (Setup)

  • 결정자: IMPALA-CNN(da=256), maze_aisc, 25M PPO. Phase 1 게이트 통과 — in-dist 성공률 0.806, OOD goal-misgen율 0.522(eligible 276).
  • 해석자: 디코더 트랜스포머(3층, d=256), 중립 코퍼스 from scratch. 3슬롯 합성언어 (AGENT_REGION 9 / HEADING 9 / CHEESE_DIR 8). describer oracle = 객관 사실 내레이터(속마음 미기재). Phase 2 게이트 통과 — slot_match 0.994, 72조합 생성 1.0, 오토인코딩 0.987. LM에는 prior가 없다 → 편향(goal-misgen)은 오직 에이전트 안에만 산다(SPLIT-9式 오염 원천 봉쇄).
  • ACC: 선형 W + LayerNorm 정규화(affine=False) + 양방향 MSE. (C-thin) 이중 grad 경계 — 에이전트는 항상 detach(오염 없음), LM은 언어 코어 stop-grad·인터페이스만 적응.
  • 빌드(같은 RL 에이전트 공유 → V2 vs B4 문자 그대로 통제): B1 단독(상한 reference) / B3 probe MLP(정보 존재 reference) / B4 Flamingo cross-attn 어댑터(K=16 latent, next-token only) / V2 ACC(분리 재구성 only).
  • 측정: #1 성능 sanity, #2 cosine + per-slot 충실도, #3 활성 스왑(인과), #4 Procrustes(미실행). Scenario A/B/C + 정량 임계치 사전 등록(§5.6/§5.8).

3. 결과 (Results)

3.1 목표 오일반화는 표상 수준에서 읽힌다

per-slot 충실도(Figure 1, Table 1)에서 cheese_dir는 in-dist 0.85(B3/B4)에서 OOD 0.07로 모든 빌드에서 붕괴한다. 반면 agent_region(위치)은 OOD에서도 0.78을 유지한다. heading은 전 빌드에서 낮다(~0.35) — 에이전트가 피드포워드 단일 프레임이라 oracle의 4스텝 궤적을 부분만 표상하기 때문이다. 핵심은 에이전트가 OOD에서 실제 치즈 방향을 표상하지 않는다는 것 — 아키텍처와 무관한 단단한 결과다. 목표 오일반화가 행동뿐 아니라 내부 표상에서 직접 관찰된다.

3.2 충실 ≠ 합리화 (핵심 기여)

OOD 합리화율(생성이 실제 방향 대신 학습 prior "우상단"과 맞는 비율, eligible n=305,513)은 B4 0.50, B3 0.40, V2 0.09다(Figure 3). 표면적으로 B4는 SPLIT-9式 "둘러대기"처럼 보인다. B4−V2 = 0.41 ≥ 0.2로 사전 등록 §5.6 합리화 임계치를 방향대로 충족한다.

그러나 활성 스왑(Figure 2, Table 2)이 이 그림을 뒤집는다. hagent(A)→(B)로 보간하면 B4/B3의 생성 HEADING이 B를 강하게 따라간다(swap-following 0.83). 즉 B4/B3는 에이전트 표상을 인과적으로 충실히 추적한다. 에이전트는 OOD에서 prior("우상단")를 표상하므로 — B4의 "합리화"는 둘러대기가 아니라 목표 오일반화한 에이전트를 충실히 읽은 결과다. swap이 인과 추적을 증명한다.

재프레임 — "충실 vs 합리화"는 SPLIT-9 가정보다 미묘하다. 목표 오일반화한 에이전트를 충실히 읽으면 오일반화된 목표를 보고하게 된다. 합리화처럼 보이는 출력이 실은 충실한 reading일 수 있다 — 통역사의 잘못이 아니다. 충실도와 과제-정합성은 별개의 축이다.

3.3 ACC 요약-벡터 병목 < 분산 cross-attn Scenario C

핵심 비교 V2 vs B4: 충실도 V2 0.375 < B4 0.66, swap-following V2 0.42 < B4 0.83. 사전 등록 §5.6 "#3 V2−B4 ≥ +0.15"에 대해 실제는 −0.41(반대 방향). 판정은 Scenario C — "분리-재구성 V2가 next-token 어댑터 B4보다 충실하다"는 핵심 가설을 기각한다.

원인 진단(Table 2 천장). ACC는 문장 전체를 단일 요약 벡터 ñlm로 압축해 선형 W로 잇는다. 어댑터는 K=16 latent로 분산 cross-attention을 LM 블록에 주입한다. 단일 요약 벡터가 병목이다. 천장 분석: closed-form 선형 W의 상한 cosine=0.467이고 비선형 MLP bridge도 0.498(≈선형, test cosine는 overfit으로 하락) — 선형성은 병목이 아니다. ACC 충실도 한계는 W의 선형성이 아니라 요약-벡터 인터페이스 자체의 용량이다.

에이전트-한계 천장. cosine 1.0은 원리적으로 불가능하다. oracle 3슬롯 중 heading(단일 프레임 한계)과 cheese_dir(OOD 오일반화 prior)은 에이전트가 애초에 표상하지 않는 부분이 있다. 따라서 천장 ~0.47 cosine / ~0.39 slot은 다리(ACC)의 한계가 아니라 에이전트의 실제 표상량을 반영한다. 측정을 per-slot으로 분해해야 진짜 신호가 보이는 이유다.

3.4 통제 2×2 — 패배의 원인은 loss인가 인터페이스인가? (confound 해소)

§3.3의 V2 vs B4 비교는 두 요인이 동시에 다르다 — 학습 신호(재구성 vs next-token)와 인터페이스(단일 요약 벡터 vs K-latent 분산 cross-attn). V2의 패배가 어느 쪽 탓인지 가르기 위해 인터페이스 × loss 2×2를 구성했다: 빠진 두 칸 B4Thin(thin × next-token)과 V2Rich(rich × reconstruction)를 채워, 얼린 Phase-3 에이전트·LM 위에서 통역사만 동일 페어로 post-hoc 적합했다(PLAN §10.1 CTRL-2x2; RL 재학습 없음).

결정적 결과 — 인터페이스를 동일하게 맞춰도 재구성이 진다. 얇은 쌍(동일한 단일벡터 인터페이스): next-token(B4Thin)이 재구성(V2)을 충실도 +0.31, swap-following +0.40로 압도한다. → V2의 패배는 얇은 인터페이스 탓만이 아니라 재구성 학습 신호 자체가 약하기 때문이다. 사전 등록 CTRL-3(재구성 − next-token이 swap ≥ +0.10 그리고 per-slot ≥ +0.05)은 충족 실패(오히려 강하게 반대) → 핵심 가설은 더 단단히 기각된다(Scenario C 강화).

인터페이스도 효과 있다. next-token을 고정하면 굵은 분산 인터페이스(B4)가 얇은 것(B4Thin)을 충실도 +0.19 / swap +0.21로 능가한다. → 두 손잡이 모두 B4의 선택을 지지한다: 좋은 학습 신호 그리고 분산 인터페이스.

정직한 주의 — V2Rich는 붕괴했다 (loss↓ ≠ 성공, 재발). rich × 재구성 칸의 MSE는 1.70→0.73으로 내려갔으나 생성이 붕괴했다(충실도 ~0, swap n_readA = 9). 원인: 이 빌드의 재구성 타깃(LM의 모든 위치 히든 시퀀스)이 ill-posed다 — 그 히든은 표면 어형까지 인코딩하는데 에이전트는 표면형은커녕 heading조차 부분만 표상하므로, reconstructor가 평균으로 회귀(붕괴)하고 lm_head 디코드가 상수화된다. 따라서 rich × 재구성 칸은 이번 실험에선 무정보다(2×2 분해의 'rich loss 효과'와 평균 인터페이스 효과는 V2Rich 오염으로 해석하지 않고 clean 셀만 사용). 잘 정의된 richer-reconstruction은 후속 과제다. (sanity: post-hoc V2 0.367 ≈ co-trained V2 0.375; B4Thin cheese 0.845 ≈ co-trained B4 — 일치. 단일 RL seed.)

3.5 후속 — 공동활성 기억 뇌량(CCM): 기록만으로 절반, "차이를 기록하라" (Phase 5)

V2 기각 이후 우리는 다리를 학습하지 않는 대안을 시험했다. 두 망이 같은 장면을 받을 때 켜진 노드의 대응을 기록만 하고(공동활성 통계), 그 기록 W로 에이전트만으로 LM을 구동한다. 인터페이스·생성 경로는 B4Thin과 byte-identical(비-아핀 LN + 단일 벡터 ĥ=W·LN(h)+b → lm.generate)이되, W는 gradient가 아니라 닫힌형 통계로 채워지는 버퍼다 — "학습이 아니라 기억"이라는 정체성. 모두 frozen Phase-3 에이전트 + Phase-2 LM 위에서.

다리는 "같이 켜진 것"이 아니라 무엇이 다른가를 기록해야 한다. 날것 공동활성 W=E[y·xᵀ]=cyx+mymxᵀ는 rank-1 평균항(공통모드)에 지배되어 상수로 붕괴한다. 중심화는 그 항을 직접 빼고, 화이트닝은 역공분산이 그 고분산 공통방향을 눌러 — 서로 다른 경로로 같은 공통모드를 제거해 차이 구조를 드러낸다. 둘은 중복이라 하나면 충분, 함께 써도(ridge) 미미하게만 낫다. 피질의 뺄셈·나눗셈 정규화가 둘 다 공통모드를 누르는 것과 정합한다. [자기교정] 앞선 step1에서 우리는 "화이트닝이 결정 성분"이라 보고했으나(근거: 중심화+직교의 Procrustes가 실패), 이 깨끗한 ablation이 그것을 confound로 교정한다 — 중심화만으로도 작동하므로 Procrustes 실패는 화이트닝 부재가 아니라 강제 직교화가 특이값(스케일)을 버린 탓이다.

그리고 가장 야심찬 형태 — 두 뇌가 적응하며 "다리가 자라는가"(닫힌고리, step2)? frozen 다리를 넘어, bridge loss를 에이전트로 역전파해 두 뇌가 다리에 적응하게 했다. 두 설정 모두에서 사전등록한 "다리가 자란다"(adapted ridge swap ≥ +0.10 그리고 slot ≥ +0.05 그리고 task 유지)는 미지지다.

닫힌고리 자가성장 — 미지지 (정직한 음성, loss↓≠성공). 세게(전체 에이전트, lr 1e-4): bridge loss는 내려갔으나 task return 10→3.75 붕괴(다리는 +0.034만, 임계 미달). 살살(conv 동결 + lr 1e-5 + 앵커): return 10→8.12 유지했으나 다리가 오히려 후퇴(swap 0.445→0.394). ⇒ 세게 밀면 뇌가 깨지고, 안 깨질 만큼 살살 하면 다리가 안 자란다. 학습 중 bridge loss 하강은 held-out 충실도 상승으로 이어지지 않았다(W와 h의 공동적응이 학습 손실만 낮춤). 이 메커니즘에서 다리-readability와 task 표상은 긴장 관계다. 다른 메커니즘(W 동시 학습·양방향 적응·다리 폭 직접 성장)은 미시도(§5).

step3 — 그래서 다리를 *살아있게* 만들고 두 뇌가 마중하게 했다 (반전). step2의 음성은 기록된 W + 에이전트만 적응이라는 한 메커니즘의 한계였다. step3는 둘을 바꾼다: (i) W를 가소성(학습 파라미터)으로 풀되 기록 ridge에서 warm-start("기억이 씨앗"), (ii) LM도 마중 — 생성 경로의 디코더 블록 blocks[0]만 작은 lr로 풀고 언어 KL 앵커(frozen 참조 LM)로 언어를 보존(짜고-치기 차단). 단계적: A1=두 뇌 고정 · W만 학습(번역기 천장 control), A2=agent (gentle: conv 동결+소lr+PPO+repr 앵커)+LM blocks[0]+W 공동적응.

4. 논의 (Discussion)

SPLIT-MNIST(동질·대칭·저차원)에서 이긴 V2가 SPLIT-MAZE(이질·비대칭·고차원)에서 진 것은, 분리-재구성 원리의 실패라기보다 인터페이스 용량의 문제로 보인다. 단일 요약 벡터는 toy에선 충분했으나 IMPALA 표상 → LM 문장 임베딩 다양체를 충실히 얹기엔 좁다. 어댑터의 분산 cross-attention은 같은 정보를 여러 지점에 주입해 이 병목을 우회한다. 통제 2×2(§3.4)가 이를 확증한다: 인터페이스를 동일하게 맞춰도 재구성 신호가 next-token보다 약하므로, V2의 열세는 인터페이스 아티팩트가 아니라 학습 신호 자체의 성질이다.

가장 큰 개념적 수확은 ②다. 정렬·해석가능성 맥락에서 "충실한 통역사"가 곧 "과제-정합 보고"를 뜻하지 않는다. 목표가 오일반화된 에이전트를 충실히 읽으면 오일반화를 보고한다. 이는 SPLIT-9의 "합리화 비판"을 정밀화한다 — 합리화처럼 보이는 출력이 실은 충실한 reading일 수 있으며, 둘을 가르려면 (활성 스왑 같은) 인과 측정이 필요하다.

5. 한계 (Limitations)

  • 단일 RL seed · descriptive. paired-bootstrap · Holm–Bonferroni 통계 확정은 multi-seed(3~5) 후속(§5.7). 단 핵심 격차가 커서(swap Δ−0.41) 방향 결론은 견고할 전망.
  • #4 Procrustes(W 위치 무관)는 미실행 — multi-seed와 함께 진행할 항목.
  • heading 천장이 에이전트 메모리 부재에 묶임 — recurrent 에이전트라면 더 높은 천장이 가능(Deferred).
  • richer-reconstruction은 well-posed 재설계 필요 — 통제 2×2(§3.4)에서 rich×재구성 칸(V2Rich)은 붕괴했다(타깃 ill-posed). 잘 정의된 richer-recon으로 그 칸을 공정히 재는 것은 Deferred. (얇은 쌍이 이미 confound를 결정적으로 풀었다.)
  • CCM(Phase 5, §3.5). 기록 다리 양성(= B4의 52%)·공통모드 제거는 단일 RL seed descriptive. step3 공동적응 효과(+0.064±0.020, 5/5 양성)는 5-seed 절차 multi-seed로 확정됐고, 다른 RL 뇌 3개로의 일반화도 GENERALIZES(평균 +0.081, 3/3 양성)로 확인 — 단 해석자 LM은 공유이므로 일반화는 결정자 뇌까지이고 해석자 뇌 변동은 미시도(큐의 큐). 효과는 뇌-의존(brain 2는 null+과제붕괴; W-budget control이 그 가짜 양성을 흡수). step3 양방향은 blocks[0] 한 블록 + 단계적(A1→A2)에 한정 — 완전 양방향(처음부터 동시)은 미시도(큐). 절차 seed 4 & brain 2에서 task return이 가드(−20%)를 살짝 초과(−23%)한 점도 기록.

6. 과정과 방법론적 정직성 (Process & Methodology Honesty)

이 결과에 이르는 길은 곧지 않았다. 세 번의 post-hoc 교정을 정직하게 기록한다 — 본 프로젝트의 부수 산출물은 이 방법론 saga 자체다.

  1. POST-HOC-5 — 노이즈 floor 인정. Phase 3 게이트 G2 = 0.792 < 0.80(7 에피소드 차, 95% CI [0.756, 0.828]; Phase 1 동일 eval 0.806과 같은 노이즈 밴드). 임계치를 임의로 낮추지 않고 왜 이 proxy가 miscalibrated였는지를 박제한 뒤 PASS. 동시에 V2 recon=0을 붕괴 위험으로 플래그.
  2. POST-HOC-6 — degenerate collapse 진단·교정. Phase 4.0 진단(diagnose_v2.py)에서 recon=0이 정렬이 아니라 붕괴임을 확인 — ĥlm std≈0, cosine=1.0(상수 두 개의 함정 신호), 생성이 전부 상수. 원인: interface_proj 상수 출력 + W→0의 trivial 최소해(BYOL/SimSiam collapse 가족). fix: LayerNorm affine=False + LM 전체 동결(ACC W만 학습) = SPLIT-MNIST V2 본형. W만 재적합(RL 재실행 없이 수 분)으로 붕괴 해소, 단 cosine 0.27 = 약함.
  3. POST-HOC-7 — 천장 진단 + W untie. ceiling_v2.py에서 선형이 병목이 아님(MLP≈선형 0.47–0.50) + tied-W GD 0.27 ≪ 선형 천장 0.467 확인 → tied 양방향 제약이 생성 방향을 깎음. fix: W untie(비대칭 Wa2l/Wl2a). untied refit cosine 0.27→0.435(in)/0.396(OOD), 선형 천장에 근접. 에이전트-한계 천장 박제.

러닝. ① loss→0 ≠ 성공 — trivial 해를 반드시 의심(cosine=1.0은 함정). ② "작은 단위·검증부터" + full harness 전에 Phase 4.0 진단을 먼저 돌린 것이 큰 낭비를 막았다. ③ 사용자의 의심("이게 정말 정렬인가?")을 실험으로 검증한 패턴이 단정적 추론보다 정확했다. ④ 사전 등록 임계치는 결과 본 뒤 변경하지 않고 왜 miscalibrated였는지만 기록한다.
